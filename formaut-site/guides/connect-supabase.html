Setup guide — Step 3 of 3

Connect your Supabase account

Supabase is your database — it stores your site's content, your admin preferences, and the session history that lets Formaut remember what was built and how. Like everything else, it's in your account. Formaut never stores your data on its own servers.

1
Create a Supabase account
Go to supabase.com and sign up. The free tier gives you one project with a 500MB database — more than enough for your site.

If you already have an account, skip to step 2.
2
Find your Organization ID
Supabase organizes projects under organizations. Formaut needs your org ID to create your database project in the right place.

The easiest way to find it: look at the URL when you're in your Supabase dashboard.

supabase.com/dashboard/org/[your-org-id]/general

Copy the ID from the URL — it's the part between /org/ and /general.
3
Generate a Management API token
This token lets Formaut create and configure your database project automatically — so you don't have to set it up manually.

supabase.com/dashboard/account/tokens → Generate new token
What to fill in
Token name: Formaut Platform
Expiry: No expiry
Permissions: Leave as default (all management operations)
Important: Copy the token immediately — Supabase only shows it once. If you close the page without copying it, you'll need to delete it and generate a new one.
4
Paste both values into Formaut
Back on your setup page, paste:

1. Your Organization ID into the org ID field
2. Your Management API token into the token field

Formaut will use these to create your database automatically — you won't need to do anything else in Supabase.
Why Supabase?

Supabase is a Postgres database with a good API layer on top. Postgres is the most reliable open-source database in the world — it's been around for 35 years and runs a significant portion of the internet.

We looked at Firebase and PlanetScale. Firebase is proprietary — your data lives in Google's format. PlanetScale moved away from a free tier. Supabase is open source, has a genuinely useful free tier, and your database is standard Postgres — any developer in the world can work with it if you ever need one.

Your content, your hours, your menu, your images — all of it is in a database that you own and can export at any time.
